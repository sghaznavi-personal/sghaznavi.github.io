Tech Interviews

October 23, 2019

Hiring the right talent for your team is a key challenge faced by many organizations!

I have been involved in interviewing candidates at various levels within technology teams and decided to jot down some key interview questions to gauge prospective candidates. The answers to these questions are descriptive by intent, as it allows candidates to share their perspective.

There are no right or wrong answers, just 'more complete' / incomplete responses.



Q1. What is the main advantage of cloud deployment for large scale web application architecture?

Ans

1. Ability to scale application architecture through pay as you go model, as supposed to large upfront investment.

2. Elastic nature of cloud allowing scaling resources up or down based on app need and only paying for what's used.

3. Taking advantage of Managed Cloud services allowing teams to focus on writing core business logic and leaving runtime and infrastructure to be managed by cloud provider.



Q2. What're the main pillars of Spring Boot Framework?

Ans

1. Dependency injection/inversion of control - wherein framework injects concrete implementations during application runtime, and application can use different implementations for different scenarios.

2. AOP - Cross cutting concerns injected into the application by the framework

3. Portable service abstraction or APIs that application needs are provided by the Spring Framework through templates like, Jdbc template, jms template. etc.

4. Spring Boot added the concept of Auto configuration. It allows apps to get framework provided default configurations for different API declared by the app.



Q3. What's Serverless Architecture?

Ans

1. Its arch where core server side logic is run on stateless containers that are request / response triggered.

2. Its, also an arch where you pay for what you use and mainly provided by cloud based Architecture (or functions as a service).

Provides - PayForValue billing

3. Allows development and deployment of applications without worrying about managing server and runtime overhead



Q4. What are the key limitations of Monolithic arch and advantage of Microservices?

Ans

Monolithic apps tightly combine different flows/services into a single large application process. The downside is, If one process of the app experiences a spoke in demand, the entire arch has to be scaled up to handle the load.

Also, any individual enhancement or fix to any individual service results in entire platforms testing, and release which is quite time consuming, as code base grows and becomes more complex.

Monolithic architectures, break large applications into smaller logic sub apps, that can scale independently, and enhancements are quicker and less risky due to limited application scope.
